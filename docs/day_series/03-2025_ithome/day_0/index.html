Day0 - 前端 30 問導讀

這是 2025 鐵人賽預計要寫的系列文，雖然我還沒決定今年要不要參加，講真，去年實在是太累了...
想說今年稍微提早一些開始佛系準備，屆時可以比較輕鬆也確保文章品質，瞧瞧我去年的鐵人賽寫到後面幾乎都剩 code 了 www

這次的系列主題暫定為「前端 30 問」。說是 30 問，其實是想以前端面試為主題，一天一個面試主題。
這次會想以面試為主題的起因是因為去年的跳槽，那時跳槽成功後我稍微寫過一篇面試紀錄，大概記了一下被問過什麼技術問題。
後來在構思 2025 鐵人賽可以拿什麼主題參賽時，看到之前那篇文章，就萌生把它寫成一個系列文的想法，同時也是一種複習的機會。

本次系列文預計會有下面主題，而我預計如果有範例 code 的話會用 TypeScript 和 C# 來寫。用 TypeScript 而非 JavaScript 純粹是因為習慣，C# 則是我自己個人練習 XD

  1. HTML & CSS - 那些 web 骨架與裝潢的問題
  • [HTML] 骨架遠比想像中的有學問
    • [HTML] Block & Inline Element
  • [CSS] 裝潢總是要有油漆吧
    • [CSS] 來聊聊元素置中的各種方法
    • [CSS] CSS 的那些單位：px, em, rem, vw, vh, %
  1. JavaScript - 賦予 web 生命的力量 (10)
  • [JS] 變數的宣告與作用域
  • [JS] 方法宣告與提升
  • [JS] 同步 or 非同步？
  • [JS] 為何陣列和物件這麼重要？
  • [JS] 面試很愛考的閉包 (Closure) 到底是怎麼一回事？
  • [JS] Call by Value or Call by Reference？
  • [JS] 事件冒泡與事件捕獲其實是一條纜車
  • [JS] 為什麼面試那麼喜歡考 this？
  • [JS] bind, call, apply 三兄弟
  • [TS] JavaScript？TypeScript？
  1. API - 這是一個餐廳前台與廚房間的對話 (6)
  • [API] 改變 web 的 AJAX
  • [API] RESTful API 是什麼？
  • [API] 串接 RESTful API 的四種方法
  • [API] GraphQL 又是什麼玩意兒？
  • [API] 404 Not Found？401 Unauthorized？
  • [API] 有 API error 怎麼辦？丟垃圾桶？
  1. 框架 - Vue 跟 React 如何改變前端開發習慣 (3)
  • [Framework] Vue or React？框架改變了什麼？
  • [Vue] Vue 跟你說你該會的那些玩意兒
  • [React] React 偶數版本表示毀天滅地級別的更新！？
  1. 開發環境 - 專案實務開發時會遇到的問題比想像中的多更多 (2)
  • [Git] Git 指令真的懂嗎？
  • [Docker] Docker 真的是前端該問的問題嗎？
  1. Domain Knowledge - 身為一個前端你該懂的事 (5)
  • [DeBug] 除錯是一門學問
  • [Test] 前端到底要不要寫測試？
  • [Mode] SPA or SSR?
  • [Defender] 前端會遇到的資安風險
  • [Efficacy] 前端效能優化
